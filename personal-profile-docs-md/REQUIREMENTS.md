# 个人资料网站需求文档

> 项目名称：个人资料网站（Personal Profile Website）  
> 仓库 / 包名：`developerprofiles`  
> 当前版本：v0.1  
> 技术方向：React + TypeScript + Vite + Tailwind CSS  
> 文案语言：**全中文**  
> 参考网站：https://aashay-dev-ed.develop.my.site.com/AashayWase/s/  
> 说明：参考网站仅作为布局结构与页面组织方式参考，不直接复制原站代码、图片、文案、Logo 或其他受版权保护的素材。实现前应对照参考站验收布局结构，不要求像素级一致。  
> 关联文档：[TECHNICAL_DESIGN.md](./TECHNICAL_DESIGN.md) · [AGENTS.md](./AGENTS.md)

---

## 1. 项目背景

用户希望以一个现有个人作品集网站为原型，进行二次改造，制作一个属于自己的个人资料网站。网站主要用于展示个人身份、头像、简介、技能、项目经历、可选经历、联系方式与社交链接。

本项目采用 React + TypeScript 实现，保留参考网站的整体单页式布局结构，同时将具体资料替换为用户自己的内容。首版使用占位数据上线，头像与简历可在后期补充。

## 2. 首版冻结决策（v0.1）

| 决策项 | 首版约定 |
|---|---|
| 文案语言 | 全中文（导航、按钮、模块标题、占位文案） |
| 语言 / 框架 | TypeScript + React + Vite |
| 移动端导航 | 汉堡按钮 + 侧滑/抽屉菜单 |
| 经历模块 | **可选**；支持工作 / 实习 / 教育 / 项目经历类型，可只选一种或多种混排 |
| 项目模块 | `projects` 为空时 **隐藏整个 Projects 区块** |

| 部署 | **国内主站**：Gitee Pages `https://<user>.gitee.io/developerprofiles/`；GitHub Pages 仅作备用 |
| 内容数据 | 首版使用占位数据，上线后由用户替换 |

## 3. 项目目标

### 3.1 业务目标

1. 建立一个清晰、专业、可维护的个人资料网站。
2. 用于展示个人品牌、技能能力、项目案例和联系方式。
3. 支持后续快速替换个人资料、头像、简历、项目与社交链接。
4. 适配桌面端、平板端和移动端浏览体验。
5. **主要面向中国大陆访问**：优先部署 Gitee Pages 或国内云静态托管；GitHub Pages 作备用，不推荐 Vercel / Netlify 作为国内主站。

### 3.2 设计目标

1. 保留参考网站的整体布局结构：顶部导航、首屏介绍、个人介绍、技能、项目、经历（可选）、联系等区域。
2. 页面风格简洁、现代、偏开发者个人作品集；**默认深色主题**。
3. 视觉上突出个人姓名、职位、头像、核心能力和项目成果。
4. 页面交互以单页滚动、锚点导航和卡片展示为主。
5. 动画克制，保证加载速度和可读性。

## 5. 网站定位

本网站定位为「个人资料 + 项目作品集」类型网站，不是博客系统，也不是复杂后台系统。

### 5.1 核心用途

- 展示个人资料
- 展示技术能力
- 展示项目案例
- 提供简历下载入口（PDF，可后期添加）
- 提供联系与社交链接入口
- 提供 GitHub、LinkedIn 等外部链接

### 5.2 非目标范围

当前阶段不包含以下功能：

- 用户登录 / 注册
- 后台管理系统
- 在线编辑个人资料
- 博客发布系统
- 支付功能
- 数据库管理
- 多用户系统
- 复杂 CMS 内容管理
- 在线联系表单

## 6. 页面结构需求

网站采用单页滚动式结构：

```text
Navbar 顶部导航（桌面横栏 / 移动汉堡+抽屉）
Hero 首屏介绍
About 个人介绍
Skills 技能展示
Projects 项目展示（无数据时整段隐藏）
Experience 经历展示（可选模块，无数据时整段隐藏）
Contact 联系方式（仅链接）
Footer 页脚
```

### 6.0 参考站对照清单（结构验收）

实现与验收时对照参考站，确认以下结构存在且信息层级合理（不要求视觉抄袭）：

| 区域 | 验收要点 |
|---|---|
| Navbar | 固定顶栏、锚点跳转、移动端可收起 |
| Hero | 头像区 + 姓名职位 + 主/次 CTA + 社交图标 |
| About | 模块标题 + 长文案介绍 |
| Skills | 标签/卡片网格，可换行 |
| Projects | 多卡片，含描述与技术标签 |
| Experience | 时间线或列表式经历（本项目中为可选） |
| Contact | 邮箱与外链入口 |
| Footer | 版权与姓名 |

## 7. 功能需求

> 技术实现细节见 [TECHNICAL_DESIGN.md §6–§8](./TECHNICAL_DESIGN.md)。

### 7.1 顶部导航 Navbar

#### 功能说明

页面顶部固定导航栏，用于快速跳转到各个内容区域。

#### 内容项

- 网站标识 / 姓名缩写 / 个人品牌名
- 首页、关于、技能、项目、经历（有数据时显示入口）、联系

#### 功能要求

1. 桌面端横向展示导航菜单。
2. **移动端使用汉堡按钮 + 抽屉菜单**，点击后展开导航项，选择后关闭抽屉并平滑滚动到对应区块。
3. 点击导航项后平滑滚动到对应区块。
4. 导航栏在滚动时保持可见。
5. 当前阶段不强制滚动高亮当前区块。
6. 若 Experience 模块被隐藏，导航中不显示「经历」项。

#### 验收标准

- 点击每个导航项可以跳转到对应页面区域。
- 移动端抽屉可打开/关闭，不出现横向溢出。
- 导航栏不遮挡主要内容。

---

### 7.2 首屏介绍 Hero

#### 功能说明

首屏用于展示最重要的个人身份信息。

#### 内容项

- 个人头像（可后期添加，无头像时显示占位）
- 姓名
- 职业标题
- 简短介绍语
- 主要按钮：查看项目
- 次要按钮：下载简历（有 `resumeUrl` 时显示）
- 社交链接：GitHub、LinkedIn、Email（有值才显示）

#### 功能要求

1. 头像支持后期替换；无 `avatar` 时使用占位（首字母或默认图）。
2. 姓名、职位、简介从配置数据中读取。
3. 「查看项目」滚动至 `#projects`（模块存在时）或隐藏。
4. 简历按钮链接至 `public/` 下 PDF；无简历时隐藏按钮。
5. 社交图标仅展示 `socials` 中已配置的项；外链新标签打开。

#### 验收标准

- 替换配置后首屏内容自动更新。
- 有头像时显示清晰、不变形。
- 有简历时 PDF 可下载/打开。
- 外部链接可正常打开。

---

### 7.3 个人介绍 About

#### 内容项

- 模块标题
- 个人简介（`about`）
- 所在地（可选，`location`）
- 教育背景（可选，`education`）

#### 功能要求

1. 支持较长文本展示。
2. 文案从 `profile` 配置读取。
3. `education` 为空时不显示教育行。

#### 验收标准

- 长文本在桌面端和移动端均可读。

---

### 7.4 技能展示 Skills

#### 功能要求

1. 技能列表从配置读取，支持任意数量。
2. 移动端自动换行。
3. `skills` 为空时可隐藏模块或显示简短占位（首版建议：空则隐藏）。

---

### 7.5 项目展示 Projects

#### 功能要求

1. 项目以卡片形式展示，数据来自配置。
2. 每个项目可含：名称、描述、技术标签、预览链接、源码链接、封面图（可选）。
3. 链接按钮文案：**「预览」**（`link`）、**「源码」**（`repo`）；对应链接为空则不显示该按钮。
4. **`projects` 数组为空或不存在时，隐藏整个 Projects 模块**，导航同步隐藏。

#### 验收标准

- 新增项目数据后自动增加卡片。
- 移动端单列，桌面端多列。
- 无项目时页面无空白 Projects 区块。

---

### 7.6 经历展示 Experience（可选模块）

#### 功能说明

展示工作、实习、教育或项目相关经历；**整个模块为可选**，用户可只配置其中一种或多种类型。

#### 经历类型（`type`）

| type 值 | 含义 |
|---|---|
| `work` | 工作经历 |
| `internship` | 实习经历 |
| `education` | 教育 / 求学经历 |
| `project` | 项目型经历（如开源、课程大作业） |

#### 每条经历字段

- 类型 `type`
- 职位 / 角色 `title`
- 公司 / 学校 / 组织 `company`
- 开始日期 `startDate`（`YYYY-MM`）
- 结束日期 `endDate`（`YYYY-MM` 或 `present`）
- 描述 `description`

#### 功能要求

1. 数据从配置读取；支持多条、多种 `type` 混排。
2. **按 `startDate` 倒序排序**（见技术文档排序规则）。
3. **`experience` 为空或不存在时，隐藏整个 Experience 模块**，导航不显示「经历」。
4. 可按 `type` 显示中文标签（如「工作」「实习」「教育」「项目」）。

#### 验收标准

- 修改配置后列表自动更新且顺序正确。
- 移动端排版清晰。

---

### 7.7 联系方式 Contact

#### 功能说明

首版仅提供可点击的联系方式与外链，**不开发表单**。

#### 内容项

- Email（`mailto:`）
- GitHub、LinkedIn、个人网站等（来自 `socials`，有值才显示）

#### 功能要求

1. 全部为链接展示，无「提交」「发送」类按钮。
2. 外部链接 `target="_blank"` + `rel="noreferrer"`。
3. 移动端点击区域足够大。

---

### 7.8 页脚 Footer

- 当前年份（自动）
- 个人姓名
- 简短版权说明

## 8. 空状态与模块可见性

| 条件 | 行为 |
|---|---|
| `projects` 为空 | 隐藏 Projects 区块；导航隐藏「项目」 |
| `experience` 为空 | 隐藏 Experience 区块；导航隐藏「经历」 |
| `skills` 为空 | 隐藏 Skills 区块（首版） |
| `resumeUrl` 为空 | 隐藏 Hero「下载简历」按钮 |
| `avatar` 为空 | 显示占位头像，不报错 |
| `socials.github` 等为空 | 隐藏对应图标/链接 |
| `education` 为空 | About 中不显示教育行 |

## 9. 内容数据需求

可变资料集中在：

```text
src/data/profile.ts
```

> 字段定义与 TypeScript 类型见 [TECHNICAL_DESIGN.md §5](./TECHNICAL_DESIGN.md)。

### 9.1 个人资料字段

| 字段 | 类型 | 必填 | 说明 |
|---|---|:---:|---|
| name | string | 是 | 姓名 |
| role | string | 是 | 职业标题 |
| location | string | 否 | 所在地 |
| email | string | 是 | 联系邮箱 |
| avatar | string | 否 | 头像路径，如 `/avatar.jpg`，可后期添加 |
| resumeUrl | string | 否 | 简历 PDF，如 `/resume.pdf` |
| intro | string | 是 | 首屏简短介绍 |
| about | string | 是 | 详细介绍 |
| education | string | 否 | 教育背景（About 区展示） |
| skills | string[] | 是 | 技能列表（可为占位） |
| projects | Project[] | 否 | 项目列表；空则隐藏模块 |
| experience | Experience[] | 否 | 经历列表；空则隐藏模块 |
| socials | SocialLinks | 否 | 社交链接 |

### 9.2 项目字段（Project）

| 字段 | 类型 | 必填 | 说明 |
|---|---|:---:|---|
| title | string | 是 | 项目名称 |
| description | string | 是 | 项目描述 |
| tech | string[] | 否 | 技术栈 |
| link | string | 否 | 预览链接 → 按钮「预览」 |
| repo | string | 否 | 仓库链接 → 按钮「源码」 |
| image | string | 否 | 封面图路径 |

### 9.3 经历字段（Experience）

| 字段 | 类型 | 必填 | 说明 |
|---|---|:---:|---|
| type | `'work' \| 'internship' \| 'education' \| 'project'` | 是 | 经历类型 |
| title | string | 是 | 职位或角色 |
| company | string | 是 | 公司、学校或组织 |
| startDate | string | 是 | `YYYY-MM`，用于排序 |
| endDate | string | 是 | `YYYY-MM` 或 `present` |
| description | string | 是 | 经历描述 |

展示用时间范围可由 `startDate` / `endDate` 格式化生成（如 `2024-01 — 至今`）。

## 10. 响应式需求

### 10.1 桌面端

- 最大内容宽度建议 `max-w-6xl`（约 1152px）。
- Hero 左右双栏布局。
- 项目卡片两列或三列。

### 10.2 平板端

- Hero 可双栏或上下布局。
- 项目卡片两列。

### 10.3 移动端

- Hero 上下布局。
- 项目卡片单列。
- **汉堡菜单 + 抽屉导航**。
- 字体与按钮间距适配触摸。

---

## 11. 非功能需求

### 11.1 性能

1. 首屏加载尽量快速。
2. 图片使用压缩 WebP / JPG / PNG；头像建议 &lt; 500KB。
3. 避免无用大型依赖。
4. **Lighthouse（生产构建预览）建议目标**：Performance ≥ 85，Accessibility ≥ 90。

### 11.2 可访问性

1. 图片提供 `alt`。
2. 图标链接提供 `aria-label`。
3. 合理标题层级；全页仅一个 `h1`（Hero）。
4. 文本与背景对比度足够。
5. 主要链接可键盘聚焦。

### 11.3 SEO（v0.1 最小集）

1. `index.html` 设置 `title`、`meta description`（中文）。
2. 正文包含真实姓名、职位、技能关键词（占位期用占位名，上线前替换）。
3. **v0.1 包含**：`favicon`、`og:title`、`og:description`；`og:image` 可在有头像后补充。

### 11.4 可维护性

1. 页面组件化。
2. 内容配置化（`profile.ts`）。
3. 样式规则统一（Tailwind）。
4. 提供项目 `README.md`：本地运行、替换资料、部署说明。

## 13. 部署需求（国内优先）

### 13.1 主站（推荐）

- **Gitee Pages**：仓库名 `developerprofiles`，访问 `https://<用户名>.gitee.io/developerprofiles/`。
- 构建：`npm run build:gitee`，发布目录 `dist/`。
- 国内访问速度与稳定性优于 GitHub Pages。

### 13.2 可选增强

- **独立域名 + 腾讯云 / 阿里云静态托管 + CDN**：使用 `npm run build:root`（`base: '/'`），需按厂商要求完成 ICP 备案。
- **GitHub Pages**：仅作代码同步与海外/备用访问，不作为国内招聘主链接。

### 13.3 构建与资源

- 默认子路径构建：`VITE_BASE=/developerprofiles/`（见 `docs/DEPLOY-CN.md`）。
- 头像、简历放在 `public/`，构建后随 `dist/` 一并发布。
